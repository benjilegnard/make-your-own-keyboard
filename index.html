La home row ?

Principe en usage efficace du clavier : Vos doigts au repos sont sur cette ligne. Pour ca qu'il y a des petits bumps sur F et J, généralement c'est là qu'on mets ses index Plus les lettres usuelles sont proche de la home row moins on fait d'effort pour l'atteindre

DVORAK

Touche usuelle de la home row. Pas vraiment de gain énorme.

BÉPO

Variante de dvorak adapté. Mon conseil: garder les lettres en azerty, on y est habituées,

KeyCodes

  • KC_LSHFT: Shift gauche
  • KC_ESC: Echap
  • Et caetera...

Source: https://github.com/qmk/qmk_firmware/blob/master/docs/keycodes.md

Layer functions

  • MO(x): Momentary turn layer on
  • TG(x): Toggle layer on/off
  • DF(x): Sets default layer
  • Pratique si vous voulez tester plusieurs dispositions

Mod key combinations

LShift(KC_X) RShift(KC_X)
LCtrl(KC_X) RCtrl(KC_X)
LAlt(KC_X) RAlt(KC_X)
LGui(KC_X) RGui(KC_X)
  • En paramètre entre les accolades on mets la touche voule pour accéder au caractère qui correspond à shift + KC_X
  • une pression = ctrl + x

Mod key combinations (2)

RSA(KC_X) (right shift + right alt + x)
MEH(KC_X) (ctrl + shift + alt + x)
HYPR(KC_X) (ctrl + shift + alt + win + x)
  • Permets de faire des combinaisons de touches modificatrices
Source: https://github.com/qmk/qmk_firmware/blob/master/docs/keycodes.md

Problèmes rencontrés

Le cé cédille

  • Existe pas en qwerty
  • Solution: utiliser le layout US intl with dead keys
  • Touches mortes: `+e=è
  • Mettre un altGR (right alt) sur son layer 0

Passer d'azerty en qwerty

  • Configurer 2 langues de saisie
  • Windows : shift + capslock
  • Linux (gnome) : super + space

Changer ses habitudes

  • Compliqué quand on tape depuis longtemps sur le même layout
  • Rien n'est insurmontable, il suffit de répéter les choses.
  • "It gets easier"
  • Mémoire musculaire, Index droit veut taper sur T, G, B et index gauche sur Y, H, N
  • 2 semaines un peu pénibles
  • Il faut apprendre à ré-apprendre
